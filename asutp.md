# ASUTP / AKB5

## Что это

`AKB5` это Windows-десктопное приложение на C# / .NET 8 WinForms для ведения древовидной базы знаний по АСУТП.

Проект хранит структуру по цехам, отделениям, оборудованию и связанным объектам. Основной внутренний формат хранения на текущем этапе это JSON.

## Что умеет приложение

- хранить древовидную структуру объектов
- вести несколько цехов
- искать по дереву
- добавлять, удалять и переименовывать узлы
- копировать и вставлять узлы
- перемещать узлы через drag-and-drop

- сохранять базу через `Save As`

## Что было найдено в исходной версии

Ключевые проблемы исходной выгрузки до правок:

1. Был риск потери данных, потому что корректное полное сохранение не выполнялось при обычном закрытии.
2. `paste` и `drag-and-drop` некорректно работали с поддеревьями и глубиной уровней.
3. Reload выполнялся без безопасного dirty-check.
4. Undo-история ограничивалась неправильно и теряла не те снапшоты.
5. Инициализация пустой базы была хрупкой.
6. Почти вся логика была сосредоточена в `MainForm.cs`.
7. Поиск мог устаревать после мутаций дерева.
8. Техническая гигиена проекта была слабой: артефакты сборки, отсутствие тестов, отсутствие нормальной документации и build-контура.

## Решения по развитию

### JSON или Excel

Excel не стоит делать основным хранилищем для этой задачи.

Рациональная схема на текущем этапе такая:

- source of truth: JSON
- user-facing exchange format: Excel

Значит Excel разумно добавлять как импорт/экспорт, но не как primary storage.

### Когда думать о SQLite

Переход с JSON на SQLite имеет смысл, если появятся:

- большие объёмы данных
- несколько операторов
- аудит изменений
- более сложный поиск и фильтрация
- более строгие требования к устойчивости хранения

Пока JSON ещё допустим, но следующий серьёзный шаг зрелости это скорее SQLite, чем Excel как основной формат.
